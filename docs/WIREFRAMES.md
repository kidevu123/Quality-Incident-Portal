# Wireframes (text / structure)

## Agent inbox

- Header: product mark, global search (future), command palette hint, user menu
- Left: queue list (General, Logistics, Escalations), saved filters, macro picker
- Main: dense table — ticket id, subject, account, status badge, priority, SLA clock, assignee
- Footer: pagination, bulk actions (future)

## Ticket workspace (three columns)

- Left (narrow): queue context, status dropdown, quick links to quality/RMA
- Center (wide): threaded messages (public vs internal styling), composer with internal toggle
- Right (medium): **Customer 360** summary, claim card, batch snippet, reimbursement, **Zoho SO** CTA, sync log strip

## Customer 360

- Hero: account name, claim rate, volume, abuse score, notes
- Two columns: open tickets list, recent claims table
- Future: charts, repeat-issue tags, Zoho AR snapshot

## Distributor portal

- Marketing-quality header; home CTA; claim wizard (multi-step future)
- Claim history table; ticket thread (customer-visible messages only)

## Investigation / CAPA

- Incident header + batch
- Related claims table; investigation timeline; CAPA fields (containment, corrective, preventive)

## Executive dashboard

- KPI cards: exposure, SLA breaches, open pipeline
- Tables: claims by SKU, hot batches
- Future: manufacturer scorecards, cohort trends

## Command palette (implemented)

- ⌘K: jump to Inbox, Dashboard, Portal, Admin
