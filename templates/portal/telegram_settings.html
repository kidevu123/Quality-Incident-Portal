Telegram claim alerts

Link Telegram to get pinged when a distributor submits a claim. Support Agents, Quality Managers, Administrators, and superusers who link here are notified in a private chat with the bot. Your server can also broadcast the same alerts to a team supergroup so everyone sees one thread and can discuss the claim there (see below).

Team group inbox (optional)

Create a Telegram supergroup, add {% if telegram_bot_username %}@{{ telegram_bot_username }}{% else %}your bot (set TELEGRAM_BOT_USERNAME){% endif %}, then put the group’s numeric chat ID in TELEGRAM_CHAT_IDS in the server .env (comma-separated; supergroup IDs are usually negative, e.g. -1001234567890). Restart the stack. New claim alerts and “👀 on it” pings are copied into that group, and your team can reply in the group to coordinate—Telegram keeps that conversation; Nexus only picks up 👀 / 👁 when someone replies to the bot’s alert or uses the ticket id (same as DMs).

For 👀 reactions on alerts in a group, Telegram usually requires the bot to be an administrator in that group. {% if telegram_broadcast_chat_count %}This server has {{ telegram_broadcast_chat_count }} broadcast recipient(s) configured in TELEGRAM_CHAT_IDS.{% else %}No TELEGRAM_CHAT_IDS are set yet—only linked staff DMs will receive alerts.{% endif %}

Staff: “I’m on this claim” (Telegram + workspace)

In the ticket workspace, put 👁 or 👀 in a reply or internal note to ping the team on Telegram, add the acceptance note, move New → Open when needed, and assign yourself if the ticket has no assignee.

In Telegram, react with 👀 / 👁 on a claim alert, or reply with those emojis (or include the ticket id). The bot webhook must include message_reaction in allowed_updates — run python manage.py telegram_set_webhook on the server after deploy.

Telegram “on it” pings and the auto internal note are rate-limited to about once per 10 minutes per person per ticket; status and assignee still update every time.
